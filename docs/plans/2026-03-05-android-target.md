# Android Target Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Enable Android as a Tauri build target with GitHub Actions CI producing unsigned APKs, and force SD quality on Android devices.

**Architecture:** Run `tauri android init` to scaffold the Android project, add an Android job to the existing GitHub Actions workflow using manual `npx tauri android build`, and add an Android platform check in `quality-tier.ts` to force SD (same pattern as the existing iOS force).

**Tech Stack:** Tauri v2, Gradle, Android SDK/NDK, GitHub Actions, Rust cross-compilation

The existing `quality-tier.ts` has `isIOSDevice()` that forces SD. Add the same for Android.

**Files:**

- Modify: `src/modules/quality-tier.ts:41-48` (add Android check in `detectDeviceTier()`)
- Test: `src/modules/__tests__/quality-tier.test.ts`

**Step 1: Write the failing test**

Add to the existing test file, in the same describe block that tests iOS detection:

```typescript
it('forces SD for Android devices', () => {
    Object.defineProperty(navigator, 'userAgent', {
        get: () => 'Mozilla/5.0 (Linux; Android 14; Pixel 8) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Mobile Safari/537.36',
        configurable: true
    });
    // Give it high scores on everything else — Android should still force SD
    Object.defineProperty(navigator, 'deviceMemory', { get: () => 12, configurable: true });
    Object.defineProperty(navigator, 'hardwareConcurrency', { get: () => 8, configurable: true });
    Object.defineProperty(screen, 'width', { get: () => 1080, configurable: true });

    const result = detectDeviceTier(mockGl as any);
    expect(result).toBe('sd');
});
```

**Step 2: Run test to verify it fails**

Run: `npx vitest run src/modules/__tests__/quality-tier.test.ts`
Expected: FAIL — the Android device scores high enough for HD because the only mobile penalty is 1 point on the UA check.

**Step 3: Write minimal implementation**

In `src/modules/quality-tier.ts`, add `isAndroidDevice()` after `isIOSDevice()`:

```typescript
/**
 * Detect if the device is running Android.
 */
function isAndroidDevice(): boolean {
    return /Android/i.test(navigator.userAgent);
}
```

Then in `detectDeviceTier()`, add the Android check right after the iOS check (after line 48):

```typescript
    if (isAndroidDevice()) {
        log.info(`Device tier detected: ${QUALITY_TIER.SD} (Android — forced SD)`);
        return QUALITY_TIER.SD;
    }
```

### Task 3: Add Android job to GitHub Actions workflow

Add an Android build job to the existing `tauri-build.yml`.

**Files:**
- Modify: `.github/workflows/tauri-build.yml`

**Step 1: Add the Android job**

Add a new job `build-android` after the existing `build` job. Do NOT add Android to the existing matrix (it needs different runners, tools, and steps).

```yaml
  build-android:
    runs-on: ubuntu-latest
    permissions:
      contents: write

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Java 17
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17

      - name: Setup Android SDK
        uses: android-actions/setup-android@v3

      - name: Install Android NDK
        run: sdkmanager --install "ndk;27.2.12479018"

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install npm dependencies
        run: npm install

      - name: Install Rust stable
        uses: dtolnay/rust-toolchain@stable
        with:
          targets: aarch64-linux-android,armv7-linux-androideabi

      - name: Rust cache
        uses: swatinem/rust-cache@v2
        with:
          workspaces: './src-tauri -> target'

      - name: Set Android environment variables
        run: |
          echo "ANDROID_HOME=$ANDROID_HOME" >> $GITHUB_ENV
          echo "NDK_HOME=$ANDROID_HOME/ndk/27.2.12479018" >> $GITHUB_ENV

      - name: Build Android APK
        run: npx tauri android build --apk --target aarch64 --target armv7

      - name: Determine release tag
        id: release_tag
        run: |
          if [[ "${{ github.ref_type }}" == "tag" ]]; then
            echo "tag=${{ github.ref_name }}" >> "$GITHUB_OUTPUT"
          else
            echo "tag=v$(jq -r .version src-tauri/tauri.conf.json)" >> "$GITHUB_OUTPUT"
          fi

      - name: Upload APK to release
        uses: softprops/action-gh-release@v2
        with:
          tag_name: ${{ steps.release_tag.outputs.tag }}
          draft: true
          files: |
            src-tauri/gen/android/app/build/outputs/apk/universal/release/app-universal-release-unsigned.apk
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**Step 2: Validate YAML syntax**

Run: `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/tauri-build.yml'))" 2>&1 || echo "YAML invalid"`
Expected: No error output (or use `npx yaml-lint` if available)

**Step 3: Commit**

```bash
git add .github/workflows/tauri-build.yml
git commit -m "ci: add Android APK build job to tauri-build workflow

Builds unsigned APKs for aarch64 + armv7 on ubuntu-latest.
Uses Java 17, Android SDK/NDK, and Rust cross-compilation targets.
APK uploaded to draft release alongside desktop artifacts."
```

**Step 3: Monitor the build**

Watch the GitHub Actions run. The Android job should:
1. Install Java 17, Android SDK, NDK
2. Install Rust with Android targets
3. Run `npx tauri android build --apk`
4. Upload APK artifact to the draft release

**Step 4: If the build fails**

Common failure points:
- **NDK version mismatch**: Update the NDK version string in the workflow to match what's available in `sdkmanager --list`
- **Missing SDK components**: Add `sdkmanager --install "platforms;android-34" "build-tools;34.0.0"` step
- **APK output path wrong**: Check actual output path with `find src-tauri/gen/android -name "*.apk"` and update the upload glob
- **Rust target compilation errors**: Check if `tauri-plugin-dialog`/`fs`/`shell` support Android (they should in Tauri v2)
