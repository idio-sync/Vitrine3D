# Splat LOD Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make the SD/HD quality toggle work for Gaussian splats by enabling Spark 2.0's LOD tree, and precompute LOD-ordered SPZ at archive export time so kiosk viewers load fast.

**Architecture:** Two layers — (1) runtime LOD via `lod: true` on every SplatMesh load (fallback, always works), (2) export-time LOD generation that transcodes splats to LOD-ordered SPZ before archiving (fast kiosk loads). A checkbox in the editor controls export-time LOD.

**Tech Stack:** Spark.js 2.0 preview (`@sparkjsdev/spark`), `PackedSplats.createLodSplats()`, `transcodeSpz()`, `SplatMesh({ lod: true })`.

**Step 2: Add the LOD generation logic in prepareArchive**

In `src/modules/export-controller.ts`, find the splat export block (lines 263-277). Replace the block starting at line 263 with LOD-aware logic:

```typescript
    if (includeSplat && assets.splatBlob && state.splatLoaded) {
        const fileName = document.getElementById('splat-filename')?.textContent || 'scene.ply';
        const position = splatMesh ? [splatMesh.position.x, splatMesh.position.y, splatMesh.position.z] : [0, 0, 0];
        const rotation = splatMesh ? [splatMesh.rotation.x, splatMesh.rotation.y, splatMesh.rotation.z] : [0, 0, 0];
        const scale: [number, number, number] = splatMesh ? [splatMesh.scale.x, splatMesh.scale.y, splatMesh.scale.z] : [1, 1, 1];

        // Export-time LOD: transcode to LOD-ordered SPZ if enabled and format supports it
        const splatExt = fileName.split('.').pop()?.toLowerCase() || '';
        const lodCompatible = ['spz', 'ply', 'splat', 'ksplat'].includes(splatExt);
        let splatBlobToArchive = assets.splatBlob;
        let archiveFileName = fileName;

        if (state.splatLodEnabled && lodCompatible) {
            try {
                log.info('Generating splat LOD for archive export...');
                deps.ui.showLoading('Generating splat LOD...', true);

                const fileBytes = new Uint8Array(await assets.splatBlob.arrayBuffer());
                const fileType = splatExt as string;
                const { fileBytes: spzBytes } = await transcodeSpz({
                    inputs: [{ fileBytes, fileType }],
                });

                splatBlobToArchive = new Blob([spzBytes], { type: 'application/octet-stream' });
                // Change extension to .spz for the archive
                archiveFileName = fileName.replace(/\.[^.]+$/, '.spz');
                log.info(`Splat LOD generated: ${fileBytes.length} -> ${spzBytes.length} bytes (${archiveFileName})`);
            } catch (lodError: any) {
                log.warn('Splat LOD generation failed, using original file:', lodError.message);
                notify.warning('Splat LOD generation failed — exporting original file.');
            }
        } else if (state.splatLodEnabled && !lodCompatible) {
            log.info(`Splat LOD skipped: .${splatExt} not supported (use .spz or .ply)`);
        }

        log.info(' Adding scene:', { fileName: archiveFileName, position, rotation, scale });
        archiveCreator.addScene(splatBlobToArchive, archiveFileName, {
            position, rotation, scale,
            created_by: metadata.splatMetadata.createdBy || 'unknown',
            created_by_version: metadata.splatMetadata.version || '',
            source_notes: metadata.splatMetadata.sourceNotes || '',
            role: metadata.splatMetadata.role || ''
        });
    }
```

Note: `transcodeSpz` reads the splat file, decodes all splats, and re-encodes them as SPZ. The splats are reordered during this process. When the kiosk loads this SPZ with `lod: true`, the LOD tree rebuilds much faster because the data is already in a spatially coherent order.

### Task 7: Remove the raw splat count cap

Now that LOD trees handle quality-tier differentiation properly, the raw `numSplats` truncation for `.sog` files is no longer needed — it produces worse results than letting the LOD tree handle it.

**Files:**

**Step 1: Remove the splat count cap**

In `src/modules/file-handlers.ts`, remove or comment out the `getMaxSplatCount()` function (lines 63-73) and remove the cap logic in `createSplatMesh` (lines 88, 92-97). The function should become:

```typescript
async function createSplatMesh(url: string, fileType?: string): Promise<any> {
    if (fileType === 'pcsogszip') {
        const response = await fetch(url);
        const fileBytes = new Uint8Array(await response.arrayBuffer());
        const decoded = await unpackSplats({ input: fileBytes, fileType: 'pcsogszip' }) as any;
        const packedSplats = new PackedSplats(decoded);
        // Build LOD tree so lodSplatCount does intelligent selection instead of truncation
        try {
            await packedSplats.createLodSplats({ quality: false });
            log.info(`LOD tree built for ${packedSplats.numSplats} splats (sog)`);
        } catch (lodErr) {
            log.warn('LOD tree computation failed, falling back to no-LOD:', lodErr);
        }
        return new SplatMesh({ packedSplats });
    }
    // For other formats (.ply, .spz, .splat), Spark decodes internally with LOD.
    return new SplatMesh({ url, lod: true, ...(fileType && { fileType }) });
}
```

Also remove the now-unused imports `resolveQualityTier`, `QUALITY_TIER`, and `getLodBudget` if they're only used by `getMaxSplatCount`. Check with grep first — they may be used elsewhere in the file.
